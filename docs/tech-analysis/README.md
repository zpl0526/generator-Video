## 写作约定

- **从入口讲起**：先说在哪个文件、哪个函数被触发，再展开内部细节。
- **关键代码贴片段**：用 `file_path:line_number` 标注源码位置，方便点击跳转。
- **数据流优先**：每个步骤回答"输入是什么 → 经过哪些转换 → 输出到哪里"。
- **末尾给观察 / 风险点**：技术分析不只是描述，要指出潜在 bug、重复代码、可优化项。
- **附文件索引**：每篇结尾列出涉及的关键文件，便于二次查阅。
